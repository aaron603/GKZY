# 2026 高考志愿预选项目说明

本项目用于在高考出分前后，持续维护“学校 + 专业 + 分数位次 + 招生计划 + 风险核验”的本地资料库，并生成给家庭讨论使用的 HTML 页面。

当前家庭画像：陕西物理类，估分 640-660，下探关注 630；本科以工科为主，兼顾后续生物医药/医工交叉读研可能；优先 985 和头部 211；中外合作办学降到最低优先级，只作为普通类方案之外的单独备选池核验。

## 推荐打开顺序

1. `2026志愿决策面板.html`
   日常主入口。用于筛选、排序、输入估分、查看候选专业、自动预填模拟志愿表、查看 2026 计划状态和待问招生办问题。

2. `2026高考志愿预选报告.html`
   家庭讨论版总报告。适合给家里人整体阅读，包含当前推荐逻辑、分数段分析、考试难度/大小年、专业方向建议、家长工作清单摘要。

3. `家长出分前后行动手册.html`
   家长行动手册。用于指导出分前后具体做什么、查哪些官方入口、怎么联系招生办、怎么记录证据。

4. `候选专业课程审计.html`
   专业课程审计。用于判断专业到底学什么，重点看数学、编程、电路信号、物理力学、生物医药衔接等课程强度。

5. `历年专业分数与招生参考.html`
   历年分数与招生参考。用于按学校查看已入库的专业最低分、平均分、最高分、位次、计划数、来源等级和下一步核验事项。

6. `保研与就业审计.html`
   保研与就业审计。用于查看重点学校的保研/升学/就业信息获取计划，以及未来 5-10 年专业方向就业判断。

7. `2025计划与实际录取人数审计.html`
   计划数与实际录取人数审计。用于区分“计划招生人数”和“最后实际录取人数”，避免把计划数误当实录人数。

8. `招生章程专业录取规则审计.html`
   招生章程规则审计。用于逐校查看进档考生专业安排原则、是否设专业级差、三大主科/外语约束和性别约束；未核原文的学校按经验默认并标待核。

9. `陕西一分一段与位次对照.html`
   陕西一分一段与位次对照。用于查看 2025 普通物理、2024/2023 理工的一分一段表、关键分数位次锚点和趋势图；2026 官方数据发布后会随更新脚本自动纳入。

10. `滨河高级中学招生行程跟踪.html`
   高校招生行程跟踪。用于筛选“西安铁一中滨河高级中学/滨河高级中学/铁一滨河”等关键词命中的明确行程，并单列相关招生组线索；随数据更新脚本同步刷新。

## HTML 文件作用

| 文件 | 主要用途 | 适合谁看 | 什么时候用 |
|---|---|---|---|
| `2026志愿决策面板.html` | 交互筛选、候选排序、模拟志愿表、同步最新数据入口 | 主要决策人 | 每次更新数据后、出分后重算时 |
| `2026高考志愿预选报告.html` | 总结性分析报告，统一家庭讨论口径 | 全家 | 阶段性讨论、给家里人快速了解全局 |
| `家长出分前后行动手册.html` | 出分前后行动清单、官方入口、招生办问询模板 | 家长 | 现在到正式填报结束前持续使用 |
| `历年专业分数与招生参考.html` | 按学校汇总专业分数、位次、计划数、来源和风险核验 | 家长和负责核数据的人 | 比对近年分数、计划增减和大小年时 |
| `候选专业课程审计.html` | 专业课程画像和课程强度审计 | 孩子和家长 | 对专业是否适合进行复核时 |
| `保研与就业审计.html` | 保研、读研价值、就业趋势、待查数据 | 家长 | 判断专业长期价值和读研必要性时 |
| `2025计划与实际录取人数审计.html` | 2025 计划数、实录人数、缺口和来源审计 | 负责核数据的人 | 补齐官方计划/实录数据时 |
| `招生章程专业录取规则审计.html` | 逐校核招生章程录取规则、专业级差、单科/外语和性别限制 | 家长和负责核章程的人 | 确认最终志愿风险、联系招生办前 |
| `陕西一分一段与位次对照.html` | 2025/2024/2023 一分一段、位次锚点和图表对比 | 家长和负责核数据的人 | 出分前估算、出分后用2026真实位次替换时 |
| `滨河高级中学招生行程跟踪.html` | 筛选候选高校是否有铁一滨河/滨河高级中学相关招生行程或招生组线索 | 家长和负责联系招生组的人 | 安排现场咨询、确认是否能在目标高中直接问招生办时 |

## 页面之间的关系

`2026志愿决策面板.html` 是操作入口，`2026高考志愿预选报告.html` 是阶段性结论，其他页面是专项审计或数据参考。

数据流大致如下：

```text
data/*.json
  -> scripts/audit-coverage-2025.mjs
  -> scripts/audit-high-tier-coverage-2025.mjs
  -> scripts/fetch-historical-admissions.mjs
  -> scripts/evaluate-candidates.mjs
  -> data/generated/candidate-evaluation.json
  -> scripts/generate-*.mjs
  -> *.html
```

专项页面会反哺 dashboard 和 report：

- 课程审计结果进入候选专业展开详情和总报告课程摘要。
- 历年专业分数与招生参考页承接 dashboard 候选排序，用于集中查看分数、计划数、位次和来源等级。
- 保研就业审计进入 dashboard 的“保研与就业趋势”和 report 的摘要。
- 计划/实录审计进入 dashboard 的“2025 计划与实录人数”和 report 的计划口径说明。
- 滨河招生行程页进入 dashboard 的候选专业工具链接，用于出分前后跟踪能否就近参加高校咨询。
- 家长行动手册与 report 的“家长工作清单”保持一致。

## 如何打开

直接双击 HTML 可以查看静态内容，但“同步最新数据”按钮不能写回本地文件。

建议使用本地服务模式：

```powershell
npm run dashboard:serve
```

然后打开：

```text
http://127.0.0.1:4173/
```

在这个模式下，`2026志愿决策面板.html` 里的“同步最新数据”按钮会执行本地更新流水线。

## 常用更新命令

只重新生成 dashboard：

```powershell
npm run dashboard
```

只重新评分候选专业：

```powershell
npm run evaluate
```

生成总报告：

```powershell
npm run report
```

生成所有 HTML 和中间数据：

```powershell
npm run build
```

`npm run build` 当前会依次执行：

1. 联网检查 2026 招生计划状态
2. 更新一分一段位次表
3. 更新滨河招生行程
4. 执行 630-660 覆盖审计
5. 执行高水平院校全量覆盖审计
6. 抓取可自动获取的 2023-2025 历年专业分数
7. 重算候选专业评分
8. 生成计划/实录审计
9. 生成历年专业分数与招生参考
10. 生成总报告
11. 生成 dashboard
12. 生成课程审计
13. 生成保研就业审计
14. 生成家长行动手册

## 使用边界

当前页面用于出分前预选和出分后快速重算，不替代最终人工核验。

正式填报前，每个志愿项必须至少复核：

1. 2026 陕西招生计划目录中的院校专业组、专业代码、计划数。
2. 学校 2026 招生章程中的专业录取规则、调剂规则、专业清/分数清/级差。
3. 专业备注、校区、学费、中外合作证书和培养地点。
4. 大类分流、转专业政策、保研就业和读研出口。
5. 体检、色觉、视力、单科成绩、外语语种等限制。

## 后续维护原则

1. 官方源优先，第三方数据只做初筛。
2. 2025 录取数据和 2026 招生计划分开维护，不混用。
3. 分数只是出分前临时口径，出分后应以真实位次为主。
4. 每次新增学校或专业，尽量同时补齐来源链接、计划数、分数位次、风险备注。
5. dashboard 用于操作，report 用于共识，专项审计页用于把关键风险讲清楚。
