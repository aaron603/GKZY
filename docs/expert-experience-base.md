# 本地志愿填报经验库 v1

更新时间：2026-06-10

## 边界说明

本经验库不是复制任何个人或机构的付费课程，也不能承诺“确保最佳”。它做的是三件事：

1. 汇总公开可引用的志愿填报方法论。
2. 把公开专家观点抽象成可复核的判断规则。
3. 结合本家庭偏好和官方数据，形成可迭代的本地规则库。

公开专家的价值在于提醒家长重视就业、专业底座、家庭资源和信息差；风险在于单一专家观点可能受行业周期影响，不能替代官方数据和校方核实。

## 公开专家观点的可吸收部分

### 1. 重视就业出口，但不要被短期热门绑架

可吸收：专业选择要看就业、升学、产业位置和家庭资源，不能只看学校名气。

需要修正：就业判断有行业周期。土木、房地产、新闻等公开争议说明，任何专家都可能在某个周期内判断偏差。因此本项目优先选择更具迁移性的技术底座：计算机、电子信息、自动化、集成电路、软件、数据、医工交叉。

### 2. 不只问“能不能上”，要问“上了以后读什么”

可吸收：强校冷门专业如果孩子不接受，风险很高；专业调剂、大类分流、校区和学费都可能影响实际体验。

落地规则：每个志愿项必须有“可接受专业底线”和“不可接受调剂风险”。

### 3. 家庭资源和城市资源要纳入判断

可吸收：就业和实习不只由学校决定，也受城市、校友、家庭资源、产业集群影响。

本项目落地：西安本地资源线和外地强专业/强城市线并行。西交、西工大、西电保留高权重；北京、成都、南京、武汉、长沙、重庆、大连等城市按专业平台纳入。

### 4. AI 时代更看重底层能力

可吸收：AI 会放大基础差异，不会让基本功变得不重要。

本项目落地：计算机/软件/AI 工程不是只看“AI”名称，而看数据结构、操作系统、网络、数据库、工程项目、数学基础、软硬结合和架构能力。

## 本家庭专属判断

| 方向 | 当前权重 | 原因 |
|---|---:|---|
| 计算机/软件/AI/数据/网安 | 高 | 孩子愿意把编程作为核心能力；AI 时代需要工程基本功 |
| 电子信息/通信/集成电路 | 高 | 接受电路、信号、控制；本地西电和外地成电/北邮强 |
| 自动化/机器人/智能制造/测控 | 中高 | 工程应用强，和制造业升级、机器人、工业AI相关 |
| 医工交叉/生物医学工程 | 中高 | 承接生物兴趣，适合本科工科 + 研究生交叉 |
| 低空/航空航天/无人系统 | 中 | 有平台和政策热度，但课程硬、周期波动大 |
| 科技艺术/交互/数字媒体 | 辅助 | 画画是个人兴趣，不作为本科主线 |

## 评分模型 v1

暂用 100 分制，用于候选专业排序的初筛，不替代人工判断：

| 维度 | 权重 |
|---|---:|
| 学校平台 | 18 |
| 专业匹配 | 24 |
| 分数/位次安全 | 22 |
| 来源质量 | 8 |
| 计划可靠性 | 6 |
| 城市和家庭资源 | 8 |
| 读研/交叉灵活性 | 8 |
| 风险控制 | 6 |

扣分项：

1. 中外合作：默认扣分，除非学校和专业优势明显。
2. 临床医学/护理：本项目默认排除。
3. 法学、财经、新闻等文科主线：本项目默认降权。
4. 大类/试验班：不是扣死，但必须核实分流规则。
