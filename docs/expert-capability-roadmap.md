# 志愿填报专家能力建设路线图

更新时间：2026-06-10

## 1. 目标

本项目的目标不是做一个简单的“按分数推荐学校”的工具，而是逐步形成类似资深志愿顾问的判断体系：

1. 数据可信：分数、位次、招生计划、专业备注、校区、学费、体检限制都有来源等级。
2. 判断可解释：每个推荐都有“为什么适合、哪里有风险、需要核实什么”。
3. 更新快速：成绩、一分一段、招生计划出来后，可以在短时间内重算候选池。
4. 家庭可讨论：报告和界面能让家里人看到同一套事实，而不是依赖口头转述。

## 2. 能力沉淀方式

### 2.1 现阶段优先用项目数据和脚本

当前最有效的沉淀方式是继续扩展现有结构：

| 层级 | 文件/模块 | 作用 |
|---|---|---|
| 数据层 | `data/*.json` | 固化家庭偏好、学校清单、专业基线、计划状态、风险规则 |
| 抓取层 | `scripts/update-data.mjs` | 定期检查官方接口是否发布 2026 陕西物理类分专业计划 |
| 报告层 | `scripts/generate-report.mjs` | 生成家人可分享的 `2026高考志愿预选报告.html` |
| 决策层 | 后续新增 scoring/ranking 模块 | 根据位次差、计划变化、专业偏好、风险项生成推荐等级 |

### 2.2 建议后续创建的 Codex Skill

等当前流程稳定后，可以创建一个本地 Codex skill：`gkzy-advisor`。它应固化以下规则：

1. 优先官方源：陕西招生考试信息网、学校本科招生网、学校官方查询系统。
2. EOL、第三方平台只作初筛，不直接作为最终填报依据。
3. 所有建议必须同时给出学校层次、专业方向、位次风险、计划风险、校区/学费/体检风险。
4. 不允许只按“热门专业名称”推荐，必须追问学院、课程、分流、就业和深造。
5. 计算机/软件/AI、电子信息/通信/集成电路、自动化/机器人/智能制造、医工交叉分别使用不同评价维度。
6. 输出必须有“可报、谨慎、排除、待核实”四类状态。

这个 skill 的价值是让后续每次对话都按同一套专家流程执行，而不是重新解释偏好。

### 2.3 当前可用插件/能力建议

| 能力 | 是否需要 | 用途 |
|---|---|---|
| Browser/Web | 需要 | 查官方政策、学校招生网、就业质量报告、专业目录 |
| Spreadsheets | 建议 | 做最终志愿表、排序、筛选、家庭协同讨论 |
| Documents | 可选 | 生成正式版给家里人阅读的 Word/PDF 报告 |
| Presentations | 可选 | 如果要给家里人开一次集中讨论会，可生成 PPT |
| Figma | 暂不需要 | 除非要设计复杂产品界面 |
| Slack/Google Drive | 视家庭协作方式决定 | 若多人协作、跨设备查看，可用 Drive/Sheets |

## 3. 如何接近专家级判断

所谓专家级，不是“知道很多学校名字”，而是能把信息变成可执行决策。建议按以下模型建设。

### 3.1 建立六维评价模型

每个志愿项至少打六个分：

| 维度 | 评价内容 |
|---|---|
| 学校平台 | 985/211/双一流、学科实力、保研、企业认可、校友资源 |
| 专业底座 | 课程硬度、可迁移能力、是否支撑未来读研和跨行业 |
| 位次安全 | 2023-2025 最低位次、波动、孩子真实位次差 |
| 计划风险 | 2026 计划相比 2025 是否扩招/缩招/新增/停招 |
| 个体匹配 | 编程意愿、工科接受度、生物兴趣、画画辅助兴趣、读研意愿 |
| 现实约束 | 城市、校区、学费、中外合作、体检、转专业、大类分流 |

### 3.2 建立专业画像库

每个专业方向都要有画像，而不是只看名称：

1. 计算机/软件/AI：重点看编程训练、系统课程、项目生态、实习机会。
2. 电子信息/通信：重点看信号、电路、电磁场、通信、嵌入式和行业平台。
3. 集成电路：重点看微电子学院、实验平台、读研率、产业城市。
4. 自动化/机器人/智能制造：重点看控制、测控、机械、电气和工程实践。
5. 医工交叉/生物医学工程：重点看工程底座、生命科学接口、读研出口。
6. 低空/航空航天：重点看力学、飞控、无人系统平台和行业周期。
7. 科技艺术/交互：只作为辅助兴趣，不替代主专业底座。

### 3.3 建立“招生办问询清单”

正式填报前，对重点学校至少问：

1. 2026 陕西物理类该专业具体计划数是多少？
2. 大类招生入校后怎么分流？分流按成绩、志愿还是综合？
3. 转专业政策是什么？热门专业转入比例大概多少？
4. 专业在哪个校区？四年都在同一校区吗？
5. 学费是否有后两年变化？软件、中外合作、特殊班尤其要问。
6. 保研率、升学去向、就业去向是否能提供分学院或分专业口径？
7. 女生比例、培养强度、实验/项目机会如何？
8. 是否有体检、色觉、单科成绩、外语语种限制？

### 3.4 建立反向排除清单

对本家庭来说，以下情况默认降权或排除：

1. 不清楚分流规则的大类。
2. 强校但专业明显不可接受。
3. 只靠热门词包装、课程底座不清楚的专业。
4. 中外合作高学费但优势不明显。
5. 校区、毕业证、培养资源不清楚。
6. 与孩子长期路线冲突的医学、师范、农林、纯文科主线。

## 4. 是否需要软件界面

建议做，但不要一开始做成复杂系统。合理路线是：

1. 当前阶段：继续用 JSON + 脚本 + `2026高考志愿预选报告.html`。
2. 成绩出来前：做一个轻量本地网页面板，能筛选、排序、标注待核实项。
3. 成绩出来后：输入真实分数和省位次，自动重算冲稳保，生成最终志愿草案。
4. 最终填报前：人工复核每个志愿项，特别是计划、专业备注、校区、学费和调剂。

界面必要功能：

| 模块 | 功能 |
|---|---|
| 数据状态 | 显示哪些学校已更新 2026 计划，哪些需人工复核 |
| 位次换算 | 输入真实分数/位次后，重新计算冲稳保 |
| 专业筛选 | 按计算机、电子、自动化、医工、低空等方向筛选 |
| 风险标签 | 标注计划缩招、校区、高学费、中外合作、体检、大类分流 |
| 志愿草案 | 支持拖拽/排序/备注/家庭讨论 |
| 问询记录 | 记录给招生办打电话、咨询会、官方群回复的结果 |
| 报告导出 | 生成 HTML/PDF/Excel，给家里人统一讨论 |

不建议做的功能：

1. 不建议做“黑箱一键推荐”。
2. 不建议无来源地自动采信第三方数据。
3. 不建议只用分数差做排序。
4. 不建议依赖实时爬虫作为唯一数据来源。

## 5. 家长信息获取 SOP

### 5.1 成绩出来前

1. 建立目标校清单：西交大、西工大、西电、电子科大、北邮、大连理工、重庆大学、山东大学、四川大学、湖南大学、中南大学、北京交通大学等。
2. 关注学校本科招生网、招生微信公众号、陕西招生考试信息网、阳光高考平台。
3. 加入学校陕西招生咨询群或地区咨询群，记录群号、负责老师、电话。
4. 收集 2026 招生章程、专业目录、选科要求、校区、学费、转专业政策。
5. 准备电话咨询问题清单，不要等出分后临时想问题。
6. 把孩子真实偏好写清楚：计算机可接受、电子控制可接受、愿意读研、生物作为交叉兴趣、画画作为辅助兴趣。

### 5.2 出分当天到填报前

1. 第一时间拿到真实分数、省位次、同分人数。
2. 把 2025 位次锚点替换为 2026 真实位次。
3. 获取陕西省 2026 招生计划目录，以省考试院发布为最终口径。
4. 对每个候选专业补齐 2026 计划数、专业备注、学费、校区。
5. 给重点学校招生办打电话，只问具体问题，不问“我这个分能不能上”这种泛问题。
6. 每个志愿项保留证据：截图、电话时间、老师姓名或群内回复。
7. 最终志愿表按“可接受专业优先”检查，不为学校牌子牺牲不可接受专业底线。

### 5.3 招聘会和就业信息

家长可以提前查：

1. 学校就业信息网。
2. 学校就业质量报告。
3. 学院官网的毕业去向、保研名单、优秀毕业生去向。
4. 校园招聘单位列表，重点看是否有华为、腾讯、阿里、字节、中兴、比亚迪、国家电网、航天航空、芯片企业、医疗器械企业等。
5. 学校所在地产业结构：西安、北京、成都、南京、武汉、长沙、重庆、大连等城市的优势产业不同。

这些信息不直接决定录取，但能判断专业出口是否真实。

## 6. 是否把家长信息获取也做到界面里

建议做，尤其是“问询记录”和“证据管理”。

推荐在界面中增加：

1. 招生办电话、官网、公众号、咨询群链接。
2. 每次咨询记录：时间、对象、问题、回复、可信度。
3. 开放日/直播/咨询会日历。
4. 招生章程、专业目录、计划表、就业质量报告的文件链接。
5. 待核实清单：大类分流、转专业、校区、学费、体检限制、专业备注。

这部分的价值很高，因为出分后时间短，真正拖慢决策的往往不是算法，而是关键问题没人提前问清楚。
