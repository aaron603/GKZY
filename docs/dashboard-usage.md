# 本地志愿决策面板使用说明

更新时间：2026-06-10

入口文件：`2026志愿决策面板.html`

## 1. 当前功能

本地面板是一个静态 HTML，不需要安装前端框架，也不需要启动服务器。它会读取脚本生成时嵌入的数据，支持在浏览器里交互筛选。

已实现功能：

1. 输入预估/实际分数，动态重算候选专业专家分。
2. 预留真实省位次输入框，后续接入真实位次模型。
3. 按专业方向、分段、来源、学校/专业关键词筛选。
4. 显示候选项数量、高优先项、官方源数量、已有计划数、待核实项。
5. 展示候选专业排序：学校、城市、专业、2025 分数、位次、计划、专家分、结论、标签。
6. 点击候选行可展开评分拆解和待核实问题。
7. 展示 2026 招生计划发布状态。
8. 自动生成前 10 个重点候选的招生办问询清单。
9. 支持导出 CSV 和复制当前前 8 项摘要。

## 2. 使用命令

只重新生成面板：

```bash
npm run dashboard
```

重新评分并生成面板：

```bash
npm run evaluate
npm run dashboard
```

联网检查计划、重新评分、生成报告和面板：

```bash
npm run build
```

## 3. 当前评分含义

当前专家分按估分中位数 650 临时计算，主要反映：

1. 学校平台。
2. 专业匹配。
3. 分数安全度。
4. 数据来源质量。
5. 是否已有计划数。
6. 城市和家庭资源。
7. 读研/交叉灵活性。
8. 风险控制。

这不是最终志愿顺序。它会把“匹配度高且稳妥”的专业排在靠前位置。最终填报需要把“冲刺价值”和“稳妥价值”分开排序。

## 4. 出分后必须改造

成绩出来后，面板要增加：

1. 按真实省位次重算，而不是按分数差重算。
2. 接入 2026 陕西招生计划目录。
3. 增加计划变化：扩招、缩招、新增、停招。
4. 增加最终志愿表编辑：排序、备注、不可接受调剂项。
5. 增加问询证据管理：电话时间、老师、答复、截图链接。

## 5. 使用边界

面板负责提高效率，不替代人工核验。最终填报前每个志愿项必须核查：

1. 2026 陕西物理普通类计划数。
2. 招生章程。
3. 专业备注。
4. 校区和学费。
5. 大类分流。
6. 转专业政策。
7. 体检和色觉限制。
